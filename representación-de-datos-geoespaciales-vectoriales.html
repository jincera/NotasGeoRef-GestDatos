Capítulo 3 Representación de datos geoespaciales vectoriales

Simple Features” (Características Simples) es un estándar más amplio que se aplica a la representación de datos geoespaciales vectoriales en general, no solo en R. Simple Features en General Estándar ISO: Las “Simple Features” son un estándar formal (ISO 19125-1:2004) que describe cómo representar objetos del mundo real en computadoras, enfocándose en la geometría espacial de estos objetos. Este estándar define cómo almacenar y recuperar estos objetos de bases de datos y qué operaciones geométricas deben definirse para ellos

La representación de datos en formato sf (Simple Features) se basa en el estándar ISO 19125-1:2004, que define cómo representar y acceder a funciones simples de información geográfica.

El estándar ISO 19125-1:2004 para la representación de datos geoespaciales en formato “Simple Features” define varias operaciones geométricas básicas que se pueden realizar sobre los objetos geográficos. A continuación, se detallan las operaciones geométricas clave: Operaciones Geométricas Intersección: Intersection: Esta operación devuelve la parte común entre dos o más geometrías. Por ejemplo, si tienes dos polígonos que se superponen, la intersección devolverá el área común entre ellos. Unión: Union: Esta operación combina dos o más geometrías en una sola. Por ejemplo, si tienes dos polígonos adyacentes, la unión devolverá un polígono que incluye ambas áreas. Diferencia: Difference: Esta operación devuelve la parte de una geometría que no se superpone con otra. Por ejemplo, si tienes un polígono y otro que se superpone parcialmente, la diferencia devolverá la parte del primer polígono que no está cubierta por el segundo. Distancia: Distance: Esta operación calcula la distancia entre dos geometrías. Por ejemplo, puedes calcular la distancia entre dos puntos o entre un punto y un polígono. Contiene: Contains: Esta operación verifica si una geometría contiene completamente a otra. Por ejemplo, si tienes un polígono y un punto, la operación “contains” devolverá True si el punto está dentro del polígono. Cruce: Crosses: Esta operación verifica si dos geometrías se cruzan. Por ejemplo, si tienes dos líneas, la operación “crosses” devolverá True si las líneas se cruzan en algún punto. Disjoint: Disjoint: Esta operación verifica si dos geometrías no se superponen. Por ejemplo, si tienes dos polígonos que no se tocan, la operación “disjoint” devolverá True. Equals: Equals: Esta operación verifica si dos geometrías son idénticas. Por ejemplo, si tienes dos polígonos que tienen las mismas coordenadas, la operación “equals” devolverá True. Overlaps: Overlaps: Esta operación verifica si dos geometrías se superponen parcialmente. Por ejemplo, si tienes dos polígonos que se superponen en alguna área, la operación “overlaps” devolverá True. Touches: Touches: Esta operación verifica si dos geometrías se tocan en algún punto o línea. Por ejemplo, si tienes dos polígonos que se tocan en una esquina, la operación “touches” devolverá True. Within: Within: Esta operación verifica si una geometría está completamente dentro de otra. Por ejemplo, si tienes un punto y un polígono, la operación “within” devolverá True si el punto está dentro del polígono

The core data structure in GeoPandas is the geopandas.GeoDataFrame, a subclass of pandas.DataFrame, that can store geometry columns and perform spatial operations. The geopandas.GeoSeries, a subclass of pandas.Series, handles the geometries. Therefore, your GeoDataFrame is a combination of pandas.Series, with traditional data (numerical, boolean, text etc.), and geopandas.GeoSeries, with geometries (points, polygons etc.). You can have as many columns with geometries as you wish; unlike in some typical desktop GIS software.

Each GeoSeries can contain any geometry type (you can even mix them within a single array) and has a GeoSeries.crs attribute, which stores information about the projection (CRS stands for Coordinate Reference System). Therefore, each GeoSeries in a GeoDataFrame can be in a different projection, allowing you to have, for example, multiple versions (different projections) of the same geometry.

Only one GeoSeries in a GeoDataFrame is considered the active geometry column, which means that all geometric operations applied to a GeoDataFrame operate on this column. The active geometry column is accessed via the GeoDataFrame.geometry attribute.
